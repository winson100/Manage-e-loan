## 2. 业务背景

### 2.1. 背景及现状

本调研旨在评估东莞地区个人经营性贷款市场的潜在需求，识别当前产品供给与客户痛点之间的差距，为设计“经营e贷”这一纯信用、差异化、数据驱动的个人经营性贷款产品提供市场依据。目标是通过数据支撑，明确产品定位、客群划分及风控策略的可行性。

### 2.2. 目标

新增一款结合线上+线下流程的个人经营性贷款业务，向标准化、自动化、互联网化方向发展，提升我行业务效率和市场影响力，对于符合条件的客户，提供在线贷款的个人经营性数据类产品“经营e贷”。

#### 4.3.1. 功能描述

| **项**         | **内容**                                                                 |
|----------------|-------------------------------------------------------------------------|
| **功能说明**   | 本功能用于实现可以放款申请操作。                                         |
| **功能路径**   | 微信银行-贷款中心-经营e贷【去用款】                                     |
| **人员及权限** | 经营e贷产品客户                                                         |
| **操作步骤**   | 1. 正式额度生效后，进入“经营e贷”界面，点击【去用款】，进入我要借钱界面；<br/>2. 完成我要借钱界面信息录入，点击【下一步】，当支付方式为“受托支付”时，进入资料上传界面，完成资料上传；<br/>3. 点击【提交】，进入放款中结果页，当支付方式为“自主支付”时，直接进入放款中结果页；<br/>4. 放款中结果界面显示“提交成功，银行正在进行放款审批。”。 |
| **业务交易规则** | **经营e贷首页可用总额度计算规则**<br/>1. 当有多笔授信额度记录时，“可用总额度”字段展示多笔生效额度的汇总值；<br/><br/>**授信额度记录页面规则**<br/>2. 系统读取当前客户下所有经营e贷授信额度记录展示在页面中，每额度记录支持在可借额度范围内发起多笔放款申请；<br/>3. 当额度记录状态为“生效”时，【去用款】按钮高亮可点击，当额度记录状态为“失效”或可用额度=0时，【去用款】按钮置灰不可点击；<br/><br/>**放款申请页面规则**<br/>4. 进入放款申请页面，立即弹出“利率说明”弹框，通过读取零售信贷系统利率信息的利率类型字段获取，具体规则：<br/>   - 当利率类型为“固定利率”时，利率说明内容：“本笔贷款年利率为固定利率，年利率x%（x系统获取），在贷款期限内，该利率保持不变。”<br/>   - 当利率类型为“浮动利率”且利率调整方式为“按对月对日调整”时，利率说明内容：“本笔贷款年利率为浮动利率，在LPR基础上±x基点（±x系统获取），在LPR利率变动时，贷款银行可根据合同约定的利率调整方式按照新的同期同档次LPR利率与原点差幅度定期进行调整，不再另行通知借款人和担保人。利率调整方式为对应起息日利率调整。自起息日起每12个月调整一次利率。利率调整日为起息日在调整当月的对应日，调整当月没有起息日对应日的，则调整当月最后一日为利率调整日。”<br/>   - 当利率类型为“浮动利率”且利率调整方式为“按指定周期调整”时，利率说明内容：“本笔贷款年利率为浮动利率，在LPR基础上±x基点（±x系统获取）。在LPR利率变动时，贷款银行可根据合同约定的利率调整方式按照新的同期同档次LPR利率与原点差幅度定期进行调整，不再另行通知借款人和担保人。利率调整方式为固定日利率调整。自起息日起，首次利率调整日确定为放款后首个1月1日，并从首次利率调整日后每12个月调整一次利率。利率调整日为首次利率调整日在调整当月的对应日，调整当月没有首次利率调整日对应日的，则调整当月最后一日为利率调整日。”<br/>   - 当贷款期限为“最长期限”，由于未录入贷款期限无法确认利率信息，不弹出利率说明内容；当录入贷款期限后，微信端通过查询接口获取贷款年利率信息后，失焦时自动弹出利率说明信息；<br/>5. “借款金额（元）”字段需校验，借款金额不大于当前本笔申请的授信额度，仅可输入[1,授信额度]区间的整数，当输入金额大于授信额度时，自动调整为授信额度；<br/>6. “借款期限（月）”字段校验规则：<br/>   - 当读取贷款期限方式为“固定期限”时，读取利率信息的贷款期限字段作为微信端借款期限字段的默认值，返显不可修改；<br/>   - 当获取贷款期限方式为“最长期限”时，读取利率信息的单笔支用最长期限字段作为微信端借款期限字段的判断条件，当录入借款期限大于单笔支用最长期限时，自动调整为单笔支用最长期限；<br/>7. “还款方式”字段读取零售信贷系统利率信息的还款方式字段；<br/>8. “贷款年利率”字段读取零售信贷系统利率信息的执行利率字段；<br/>9. “还款间隔”字段默认值为“每月”，不可修改；<br/>10. “首期本金”、“首期利息”和“首期本息”字段读取零售信贷系统利率试算获取，返显不可修改；<br/>11. “放/还款账户”字段规则：<br/>   - 放还款账户由系统通过获取已加挂的东莞银行一类户卡号/账号或东莞银行社保卡，支持选择获取，默认值为空；<br/>   - 放还款账户不支持非一类户的账户类型，不展示不支持的加挂卡号/账号；<br/>   - 放还款账户状态异常校验规则：当账户状态为“挂失”、“只收不付”、“不收不付”、“暂停非柜面状态”、“账户封闭冻结”时，点击【下一步】做拦截提示，提示内容：“还款账户状态异常”；<br/>12. “支付方式”字段支持选择“自主支付”和“受托支付”，选择“自主支付”方式时，放/还款账户仅支持东莞银行一类户，选择“受托支付”方式时，还款账户支持东莞银行一类户，收款账户类型不做限制；<br/>13. “收款账户”字段规则：<br/>   - 字段展示条件：当支付方式选择“受托支付”时，展示“收款账户”、“户名”、“银行”字段；<br/>   - “收款账户”字段：手工录入，仅支持数字，支持通过校验卡bin（银行卡前6位数字）返显“银行”字段，无法通过卡bin获取归属行时，“银行”字段支持手工选取收款银行；<br/>   - 录入“收款账户”为东莞银行一类户卡号/账号时，自动返显“银行”字段值为“东莞银行”（不判断开户所属支行），“户名”字段值手工录入；<br/>   - 录入“收款账户”为他行银行卡号/账户时，自动返显“银行”字段值，“户名”字段值手工录入，展示“网点”字段，支持手工查询获取网点信息；<br/>   - 收款账户状态异常校验规则：支持东莞银行卡号/账号进行卡异常校验，当账户状态为“挂失”、“只付不收”、“不收不付”、“账户封闭冻结”、“暂停非柜面状态”时，点击【下一步】做拦截提示，提示内容：“收款账户状态异常”；<br/>**查看/加签协议《贷款用途说明书》规则**<br/>14. 展示条件：当支付方式为“自主支付”时，展示“已阅读，并同意签署《贷款用途说明书》”选择框，默认为“未勾选”状态，未勾选“已阅读，并同意签署《贷款用途说明书》”时，【下一步】按钮置灰；<br/>15. 《贷款用途说明书》支持预览含要素内容，客户未输入贷款金额进行协议预览时，系统提示“请先输入贷款金额”。满足预览条件触发预览时，微信端调零售系统提供的方法，将相关生成信息传给零售，由零售生成含要素的贷款用途声明书文件；<br/>16. 勾选协议需强制预览，未预览勾选协议弹出拦截提示“请先阅读协议在勾选”；<br/>17. 勾选协议提交时，系统后台需登记操作记录，留痕信息包括但不限于：流水号、客户姓名、证件类型、证件号码、操作类型（勾选协议并同意）、操作时间、渠道日志信息、设备ID（或有）；<br/>18. 当登记操作日志超时或失败时，界面弹框提示，提示内容“系统繁忙，请重试”，允许客户再次提交；<br/>19. 提交放款申请成功，系统调无纸化平台对《贷款用途说明书》进行电子加签并归档影像目录；<br/>**放款申请安全认证规则**<br/>20. 安全认证方式：短信验证码+卡密；<br/>21. 短信验证模板：“尊敬的客户，您正通过我行微信银行进行[【xx贷】借款申请，验证码为【xx】，请确认是本人操作”；<br/>**资料上传界面**<br/>22. 触发资料上传界面条件：当支付方式为“受托支付”时，点击【下一步】，跳转到资料上传界面；<br/>23. 支持上传多张图片，上限为100张（支持所有图片格式）；<br/>24. 当在上传界面点击【上传并提交】时，系统应校验上传图片是否为空，如为空则限制提交并提示“请上传贷款用途材料”。提交成功后，微信端将图片上传文服并提供零售信贷系统获取文件地址，通知零售信贷系统，由零售信贷系统自行获取，返回获取结果；<br/>25. 客户上传资料命名规则：客户名称+业务场景（受托支付上传资料）+随机号+放款流水（如：李度自主支付上传资料0013245324234235）。 |
